# Анализ реализации увеличения скорости в Tetracar-Vue

## Основные параметры (из `gameState.ts`)
- `BASE_SPEED = 0.8` (начальная скорость)
- `MAX_SPEED = 4.0` (предельная скорость)
- `ACCELERATION = 0.001` (базовое ускорение)
- `NITRO_MULTIPLIER = 2.0` (множитель нитро)

## Механика обновления скорости
В игровом цикле (`useAnimate.ts`) каждые ~16 мс (60 FPS) вызывается:
```ts
gameState.baseSpeed += gameState.getCurrentAcceleration();
```

`getCurrentAcceleration()` возвращает:
```ts
acceleration.value * (1 - getCurrentSpeed() / maxSpeed.value)
```
где `getCurrentSpeed()` = `baseSpeed * multiplier` (с ограничением `maxSpeed`).

## Дифференциальное уравнение
- Без нитро (`multiplier = 1`):
  ```
  dv/dt = a * (1 - v / v_max)
  ```
- С нитро (`multiplier = 2`):
  ```
  dv/dt = a * (1 - (2 * v) / v_max)
  ```

## Тип роста скорости
Решение уравнения — **экспоненциальное приближение к пределу** (exponential decay of the gap to the limit).

Конкретно:
```
v(t) = v_limit * (1 - exp(-k * t))
```
где `k = a * multiplier / v_max`.

Таким образом, скорость увеличивается быстро вначале, затем замедляется, асимптотически приближаясь к `v_max / multiplier` (или к `v_max`, если нитро активно и `baseSpeed * 2` превышает `v_max`).

## Классификация
- **Не линейная** (линейный рост был бы `v(t) = v0 + a*t`).
- **Не логарифмическая** (логарифмический рост был бы `v(t) = a * ln(t) + b`).
- **Экспоненциальная с насыщением** (exponential saturation), также известная как **логистическая кривая** (logistic curve) в случае ограниченного роста.

## Вывод
Реализованное увеличение скорости является **экспоненциальным с насыщением** (exponential saturation), что обеспечивает плавный разгон с постепенным замедлением роста по мере приближения к максимальной скорости.
## Сравнение с логарифмической зависимостью (предел 3)

Рассмотрим гипотетическую логарифмическую зависимость скорости от времени с пределом 3. Например:

```
v_log(t) = 3 * (1 - 1 / (1 + ln(1 + t / τ)))
```

где τ — масштаб времени, подобранный так, чтобы начальный рост был сопоставим с текущей реализацией.

Для сравнения возьмем текущую экспоненциальную зависимость (без нитро, v_max = 4):

```
v_exp(t) = 4 * (1 - exp(-0.001 * t / 4)) = 4 * (1 - exp(-0.00025 * t))
```

**Качественное сравнение:**

1. **Начальный рост (малые t)**:
   - Экспоненциальная: производная при t=0 равна `a = 0.001`. Скорость растёт линейно в самый первый момент.
   - Логарифмическая: производная при t=0 стремится к бесконечности (если использовать формулу выше) или зависит от параметров. Обычно логарифмический рост в начале быстрее, чем линейный, но быстро замедляется.

2. **Средний участок (t ~ 2000–5000 мс)**:
   - Экспоненциальная: скорость достигает ~2.0 за ~5000 мс (примерно).
   - Логарифмическая: при правильно подобранных параметрах может расти медленнее, так как логарифм растёт очень медленно после начального скачка.

3. **Достижение предела**:
   - Экспоненциальная: асимптотически приближается к 4, достигая 95% предела (3.8) за время ~3/k ≈ 12000 мс.
   - Логарифмическая с пределом 3: приближается к 3, но может делать это ещё медленнее, особенно если предел меньше.

**Ответ на вопрос**: Если сделать логарифмическую зависимость с пределом 3, она, скорее всего, будет расти **медленнее**, чем текущая экспоненциальная с пределом 4, особенно на средних и больших временах. Однако на очень малых временах логарифмическая может давать более резкий старт (зависит от параметров). В целом, для игрового ощущения экспоненциальная зависимость обеспечивает более плавный и предсказуемый разгон, тогда как логарифмическая может казаться «вялой» после первых секунд.

**Рекомендация**: Если цель — сделать игру более динамичной в начале, но с более низким максимальным пределом, можно рассмотреть комбинированную кривую. Однако текущая реализация (экспоненциальная с насыщением) является стандартной для гоночных игр и обеспечивает хороший баланс.
